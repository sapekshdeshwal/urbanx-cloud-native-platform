# Docker Day 3 - Interview Questions

## Q1. What is a Docker Volume?

A Docker Volume is a Docker-managed persistent storage that survives container deletion.

---

## Q2. Why do we use Docker Volumes?

To store application data permanently even if containers are recreated.

---

## Q3. What are Ephemeral Containers?

Containers are temporary and can be deleted or recreated at any time.

---

## Q4. Difference between Container Storage and Docker Volume?

Container Storage:
- Temporary
- Deleted with the container

Docker Volume:
- Persistent
- Independent of containers

---

## Q5. What are Docker Networks?

Docker Networks enable communication between containers.

---

## Q6. Name Docker's default networks.

- bridge
- host
- none

---

## Q7. Why do we create a custom Docker Network?

- Better organization
- Better security
- Application isolation
- Easy communication using container names

---

## Q8. Why use container names instead of IP addresses?

Container IP addresses change after recreation.

Docker DNS automatically maps container names to the latest IP.

---

## Q9. What is Docker DNS?

Docker's built-in service discovery mechanism that resolves container names into IP addresses.

---

## Q10. What does docker inspect do?

Displays detailed information about a container such as image, network, IP, mounts and environment variables.

---

## Q11. What does docker logs do?

Displays logs generated by the container's main process.

---

## Q12. Difference between docker logs and docker logs -f?

docker logs:
Shows existing logs once.

docker logs -f:
Continuously streams new logs in real time.

---

## Q13. Why was 'ping' not available in the Ubuntu container?

The official Ubuntu Docker image is minimal and does not include utilities like `ping`. It can be installed using:

```bash
apt install -y iputils-ping
```

---

## Q14. Why should software be installed in a Dockerfile instead of manually inside a running container?

Because containers are temporary. Changes made manually are lost when the container is deleted. Installing software in the Dockerfile ensures every new container created from the image has the same software.
